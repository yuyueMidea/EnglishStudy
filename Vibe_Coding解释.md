一、什么是 AI 编程 / Vibe Coding（一句话版）

AI 编程（Vibe Coding）本质上不是 AI 在“直接写代码”，而是：
- AI 在理解你的意图 → 规划修改 → 通过工具安全、可控地改动真实代码
- 它是一个 「大语言模型 + Agent + 工具 + 工程约束」 共同协作的系统。

二、整体架构：
```
人类（自然语言）
   ↓
AI Agent（中控）
   ↓
大语言模型（LLM：负责理解与决策）
   ↓
工具调用（函数 / 终端 / 文件 / 浏览器）
   ↓
真实代码仓库 & 运行环境
```
⚠️ 关键点：LLM 不直接操作世界、一切真实操作都必须 通过 Agent + 工具

三、AI 编程的核心原理（展开版）
- 1️⃣ 程序员提前写好「函数」（工具）
- 2️⃣ Agent 把「函数信息」打包给大语言模型
- 3️⃣ 大语言模型的真正角色：决策者，而不是执行者
- 4️⃣ 浏览网页 / 执行终端命令的本质

五、AI 编程的关键优化：上下文工程；AI编程真正强的地方：主动收集上下文信息，而不是等用户说；

六、Vibe Coding 的真正“原理总结”， 把内容压缩成一句工程真相：
- Vibe Coding 不是“让 AI 随便写代码”，而是：
- 用工程手段，把 LLM 变成一个会用工具、能被校验、受约束的“高级实习生”
